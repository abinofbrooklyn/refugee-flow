**TransitionOutlet.tsx** — Create at `src/components/router/TransitionOutlet.tsx`:

- Import `useOutlet`, `useLocation` from react-router-dom
- Import `TransitionProvider` from TransitionContext
- Import styled-components for `TransitionWrapper` and `RouteLayer`

State machine with three states: `'idle' | 'loading' | 'transitioning'`

`TransitionWrapper` styled div:
```css
position: relative;
width: 100%;
height: calc(100vh - 40px);
overflow: hidden;
```

`RouteLayer` styled div with transient props `$opacity: number` and `$isOld: boolean`:
```css
position: ${props => props.$isOld ? 'absolute' : 'relative'};
top: 0;
left: 0;
width: 100%;
height: 100%;
opacity: ${props => props.$opacity};
transition: opacity 400ms ease-in-out;
will-change: opacity;
pointer-events: ${props => props.$isOld ? 'none' : 'auto'};
```

Logic flow:
1. On mount: `state = 'idle'`, render `currentOutlet` wrapped in TransitionProvider
2. On `location.pathname` change (useEffect dep on `location.pathname`, NOT `location.key`):
   - If new pathname === previous pathname ref: skip (no-op)
   - Else: stash `currentOutlet` into `prevOutlet` state, set `state = 'loading'`
   - Store new outlet from `useOutlet()` as `currOutlet`
3. When `signalReady(loadDurationMs)` is called via context:
   - If `loadDurationMs < 100`: instant switch — set `prevOutlet = null`, `state = 'idle'`
   - If `loadDurationMs >= 100`: set `state = 'transitioning'` (triggers CSS opacity change)
4. On `onTransitionEnd` of the OLD RouteLayer (opacity reached 0):
   - Set `prevOutlet = null`, `state = 'idle'`

Rendering:
- `state === 'idle'`: Single RouteLayer with `$opacity={1}` `$isOld={false}` containing currOutlet in TransitionProvider
- `state === 'loading'`: Two RouteLayer divs stacked. Old: `$opacity={1}` `$isOld={true}` with prevOutlet. New: `$opacity={0}` `$isOld={false}` with currOutlet in TransitionProvider
- `state === 'transitioning'`: Two RouteLayer divs. Old: `$opacity={0}` `$isOld={true}` with `onTransitionEnd` handler. New: `$opacity={1}` `$isOld={false}` with currOutlet in TransitionProvider

Important: Wrap ONLY the NEW (currOutlet) RouteLayer contents in TransitionProvider — the old layer does not need signaling capability.

In `src/components/RefugeeRoute_map.tsx`, line 288, change the document-global querySelector to be scoped to the component's container ref.

**Current code (line 288):**
```tsx
const canvasEl = document.querySelector('.canvas_overlay') as HTMLCanvasElement | null;
```

**Replace with:**
```tsx
const canvasEl = containerRef.current!.querySelector('.canvas_overlay') as HTMLCanvasElement | null;
```

This prevents DOM collision when two RefugeeRoute_map instances exist simultaneously during crossfade — `document.querySelector` would return the first match (old map's canvas), while `containerRef.current!.querySelector` is scoped to THIS instance's container div.

No other changes to this file in this task. The MapLibre init gating (deferring map creation until active) is handled in Plan 02.
