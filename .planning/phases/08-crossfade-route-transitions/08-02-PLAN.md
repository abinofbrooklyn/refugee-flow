Replace the loading gate (lines 131-137):
```tsx
if (loading) {
  // During route-to-route transitions, TransitionOutlet keeps old route visible
  // so we render an invisible placeholder instead of the ScaleLoader spinner.
  // The ScaleLoader only shows on the very first /route/ visit (no old route).
  if (transitionSignal) {
    return <div style={{ height: '100vh', background: '#1a1a2e' }} />;

    </div>
  );
}
```

The invisible placeholder ensures the new route occupies space in the stacked layout but shows nothing (the old route is visible on top). Once loading finishes, the useEffect fires signalReady, TransitionOutlet crossfades, and the full new route becomes visible.

Note: The very first visit to any /route/ page (e.g., from /conflict or /landing) will still show the ScaleLoader because there is no old route to crossfade from. Only route-to-route transitions get the crossfade treatment.
