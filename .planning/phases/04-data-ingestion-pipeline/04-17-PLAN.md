# Plan 04-17: Fix Route Death Data Loss

<objective>
Remove geo precision reduction and row deduplication from the IOM ingestion pipeline for route_deaths. The precision reduction was designed for the THREE.js globe (war_events) to prevent GPU crashes, but was incorrectly applied to route deaths too. This causes real death records to be snapped to a grid and deduplicated, losing data. Re-ingest IOM data to restore full-precision coordinates.
</objective>

<tasks>

<task id="04-17-01" type="auto">
<name>Remove geo precision reduction and dedup from IOM ingestion</name>

<read_first>
- server/ingestion/iomIngestion.ts
- server/ingestion/iomNormalizer.ts
- server/controllers/api/data/helpers/dataProcessors.ts (to understand reduceGeoPercision)
</read_first>

<action>
1. In `server/ingestion/iomIngestion.ts`:
   - Remove `import { reduceGeoPercision }` line
   - Change `lat: reduceGeoPercision(lat, 2)` to `lat: parseFloat(lat)` (keep NaN filtering)
   - Change `lng: reduceGeoPercision(lng, 2)` to `lng: parseFloat(lng)`

2. In `server/ingestion/iomIngestion.ts`:
   - Remove the `deduplicateRows` call on line ~97. The DB already handles dedup via `onConflict('id').ignore()` — each IOM record has a unique ID.
   - Remove the import of `deduplicateRows` from iomNormalizer

3. Keep `deduplicateRows` function in iomNormalizer.ts (don't delete it — it may be used elsewhere or useful later)

4. Verify the NaN filter still works: rows with NaN lat/lng should still be filtered out before DB insert.
</action>

- `npx tsc --noEmit --project tsconfig.server.json` exits 0

<task id="04-17-02" type="auto">
<name>Re-ingest IOM data to restore full-precision coordinates</name>

<read_first>

</read_first>

<action>
1. Clear existing route_deaths data: `DELETE FROM route_deaths WHERE 1=1` (or truncate)
2. Trigger a full IOM re-ingest. IOM always downloads the full CSV with `onConflict('id').ignore()`, so a fresh ingest after clearing will repopulate with full-precision coordinates.
3. Run the ingestion: `node -e "require('./server/ingestion/iomIngestion').ingestIomData().then(r => console.log(r))"`
4. Verify row count is >= the previous count (should be higher since dedup no longer drops records)
5. Verify coordinates have more than 2 decimal places: `SELECT lat, lng FROM route_deaths LIMIT 5`
</action>

- No NaN values: `SELECT COUNT(*) FROM route_deaths WHERE lat IS NULL OR lng IS NULL` returns 0
