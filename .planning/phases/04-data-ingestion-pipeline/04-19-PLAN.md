# Plan 04-19: Hybrid Route Death Loading

<objective>
Add server-side route filtering to the route_death API endpoint and implement
hybrid loading on the frontend: load current route data immediately on page
render, then prefetch remaining routes in the background. First load is fast,
route switching is instant once prefetch completes.
</objective>

<task id="04-19-02" type="auto">
<name>Implement hybrid loading in frontend</name>

<read_first>
- src/utils/api.ts
- src/components/RefugeeRoute.tsx
</read_first>

<action>
1. In api.ts, add a new function:
   ```typescript
   export function get_routeDeathByRoute(route: string): Promise<RouteDeath[]> {
     const url = `/data/route_death?route=${encodeURIComponent(route)}`;
     return fetch(url).then(res => res.json());
   }
   ```

2. In RefugeeRoute.tsx:
   - On mount, fetch current route data immediately via `get_routeDeathByRoute(currentRoute)`
   - Render the map as soon as current route data arrives
   - In a separate useEffect, prefetch ALL route data via existing `get_routeDeath()` in the background
   - Once prefetch completes, replace the single-route data with the full dataset
   - Route switching uses the full dataset (instant) if prefetch is done,
     or falls back to `get_routeDeathByRoute(newRoute)` if still loading

3. Add a loading state for initial route data fetch (brief spinner)
</action>

<acceptance_criteria>
- Initial page load fetches only current route data (network tab shows `?route=X`)
- Background prefetch fetches full dataset after initial render
- Route switching is instant once prefetch completes
- If prefetch hasn't completed, switching fetches the specific route
- `npx tsc --noEmit` exits 0
- `npm run build` succeeds
</acceptance_criteria>

<verify>
<automated>npx tsc --noEmit && npm run build</automated>
</verify>
</task>
