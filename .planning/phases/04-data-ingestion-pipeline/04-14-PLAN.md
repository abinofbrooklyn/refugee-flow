# Plan 04-14: Automate UNHCR Asylum + Eurostat Monthly Asylum Ingestion

**Status:** COMPLETE (already automated in Plans 04-03, 04-04, 04-08, 04-09)
**Detailed plan:** `docs/superpowers/plans/2026-03-20-automate-remaining-data-sources.md` (Phases 3-4)

## Goal
Convert UNHCR asylum and Eurostat monthly asylum from manual seed to automated pipelines with cron scheduling.

## Tasks

### UNHCR Asylum Applications
1. Examine current `asy_applications` schema and seed logic
2. Create `scripts/ingestUNHCR.js` — REST API download, country code mapping, diff-based upsert
3. Create `scripts/updateUNHCR.js` — call API with `download=true`, unzip, run ingestion
4. Tests and update `DATA_SOURCES.md`

### Eurostat Monthly Asylum
1. Examine current Eurostat usage and quarterly estimation logic
2. Create `scripts/ingestEurostat.js` — SDMX API, monthly→quarterly aggregation, diff-based upsert
3. Create `scripts/updateEurostat.js` — fetch from API, run ingestion
4. Tests and update `DATA_SOURCES.md`

## Access Details

**UNHCR:** REST API at `https://api.unhcr.org/population/v1/asylum-applications/` — no auth, `download=true` returns ZIP with CSV, ~119K rows

**Eurostat:** SDMX API at `https://ec.europa.eu/eurostat/api/dissemination/sdmx/2.1/data/migr_asyappctzm/` — no auth, JSON/TSV format, 2008-present

## Cron
```
0 0 20 * *  node scripts/updateUNHCR.js >> logs/unhcr-update.log 2>&1
0 0 22 * *  node scripts/updateEurostat.js >> logs/eurostat-update.log 2>&1
```

## Notes
- UNHCR uses non-standard 3-letter country codes (e.g., GFR for Germany) — use `coo_iso`/`coa_iso` fields
- Eurostat quarterly estimation logic already exists from Plan 04-08/04-09 — reuse it
- UNHCR provides annual totals; Eurostat seasonal ratios distribute into quarters
