# Plan 04-13: Automate IOM Missing Migrants + Frontex IBC Ingestion

**Status:** NOT STARTED
**Detailed plan:** `docs/superpowers/plans/2026-03-20-automate-remaining-data-sources.md` (Phases 1-2)

## Goal
Convert IOM Missing Migrants and Frontex IBC from manual seed/download to automated pipelines with cron scheduling.

## Tasks

### IOM Missing Migrants
1. Examine current `route_deaths` schema and seed logic
2. Create `scripts/ingestIOM.js` — diff-based upsert by Incident ID, reuse `iomNormalizer.js`
3. Create `scripts/updateIOM.js` — download from stable CSV URL (no scraping needed)
4. Tests for coordinate parsing, date parsing, route mapping
5. Update `DATA_SOURCES.md`

### Frontex IBC
1. Create `scripts/updateFrontex.js` — scrape migratory map page for XLSX link, run existing `ingestFrontexIBC.js`
2. Tests for URL extraction regex
3. Update `DATA_SOURCES.md`

## Access Details

**IOM:** Stable CSV at `https://missingmigrants.iom.int/sites/g/files/tmzbdl601/files/report-migrant-incident/Missing_Migrants_Global_Figures_allData.csv` — no scraping, no auth

**Frontex:** Scrape `https://www.frontex.europa.eu/what-we-do/monitoring-and-risk-analysis/migratory-map/` for XLSX href — no auth, curl works

## Cron
```
0 0 5  * *  node scripts/updateIOM.js >> logs/iom-update.log 2>&1
0 0 10 * *  node scripts/updateFrontex.js >> logs/frontex-update.log 2>&1
```
