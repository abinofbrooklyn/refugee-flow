Create the pure-function modules for asylum data normalization: country name canonicalization and quarterly estimation from seasonal ratios. Fix the latent Eurostat duplicate SY key bug. Unit test both modules thoroughly.

UNHCR provides only annual totals, creating misleading Q1-only spikes in the chart. These two modules solve both problems as testable, database-free pure functions.

      - Group by origin country name
      - For each origin, sum values per quarter (q1, q2, q3, q4) across all EU destinations
      - Compute ratio per quarter = quarterSum / totalSum
      - Return object: `{ 'Afghanistan': { q1: 0.30, q2: 0.25, q3: 0.22, q4: 0.23 }, ... }`
      - If total for an origin is 0, skip that origin (don't divide by zero)

      - If ratios is null/undefined, use equal split `{ q1: 0.25, q2: 0.25, q3: 0.25, q4: 0.25 }`
      - q1 = Math.round(annualTotal * ratios.q1)
      - q2 = Math.round(annualTotal * ratios.q2)
      - q3 = Math.round(annualTotal * ratios.q3)
      - q4 = annualTotal - q1 - q2 - q3 (remainder to q4 ensures exact sum)
      - Return `{ q1, q2, q3, q4 }`
