The user must complete these prerequisite steps before deploying anything. Nothing in this phase works without them.

**Step 1: Create an AWS account (if you don't have one)**
1. Go to https://aws.amazon.com/ → "Create an AWS Account"
2. Provide email, password, account name
3. Enter payment method (required — you won't be charged more than ~$10/mo for this project)
4. Choose "Basic (Free)" support plan
5. Complete identity verification (phone/SMS)
6. Wait for account activation (can take a few minutes)

**Step 2: Create an IAM admin user (do NOT use root account for CLI)**
1. Sign into AWS Console as root → go to IAM (https://console.aws.amazon.com/iam/)
2. Users → Create user
3. User name: `refugee-flow-admin`
4. Check "Provide user access to the AWS Management Console" (optional — for console access)
5. Next → Attach policies directly → check `AdministratorAccess`
6. Create user
7. Go to the user → Security credentials tab → Create access key
8. Select "Command Line Interface (CLI)"
9. Acknowledge the warning → Create access key
10. **SAVE the Access Key ID and Secret Access Key** — you won't see the secret again

SECURITY NOTES:
- Never commit these keys to git. They go in `~/.aws/credentials` only.
- After Phase 14 is complete, you could scope down this user's permissions to only what's needed. For initial setup, admin access is simplest.
- Enable MFA on both root and this IAM user: IAM → Users → Security credentials → Assign MFA device

**Step 3: Install and configure AWS CLI**
```bash
# macOS (if not already installed)
brew install awscli

# Configure with your IAM user credentials
aws configure
```
When prompted:
```
AWS Access Key ID: [paste from Step 2]
AWS Secret Access Key: [paste from Step 2]
Default region name: us-east-1
Default output format: json
```

**Step 4: Verify CLI access**
```bash
aws sts get-caller-identity
```
Should return your account ID and user ARN. If this fails, credentials are wrong.

**Step 5: Enable MFA on root account (CRITICAL)**
1. Sign in as root → IAM → Dashboard
2. "Activate MFA on your root account" → Manage MFA
3. Choose "Authenticator app" → scan QR code with your phone
4. Enter two consecutive codes → Activate

**Step 6: Set up a billing alarm (safety net)**
1. Go to AWS Budgets: https://console.aws.amazon.com/billing/home#/budgets
2. Create budget → "Cost budget"
3. Budget name: "refugee-flow-monthly"
4. Budget amount: $15 (gives headroom above the ~$10 estimate)
5. Add alert at 80% threshold ($12) → enter your email
6. Create budget

This ensures you get emailed if costs approach or exceed expectations.

The user must perform these steps (requires AWS CLI configured from Task 0):

**Step 1: Deploy the CloudFormation stack**
```bash
aws cloudformation deploy \
  --template-file infrastructure/cloudformation.yaml \
  --stack-name refugee-flow \
  --capabilities CAPABILITY_IAM CAPABILITY_NAMED_IAM \
  --parameter-overrides AlertEmail=YOUR_EMAIL \
  --region us-east-1
```

Note: Omit DomainName parameter for initial deploy (uses CloudFront default URL). Add DomainName later when domain is registered.

Wait for stack creation to complete (~5-10 minutes). Monitor in AWS Console > CloudFormation > Stacks.

**Step 2: Confirm SNS email subscription**
Check the AlertEmail inbox for an AWS SNS subscription confirmation email. Click "Confirm subscription."

**Step 3: Populate Secrets Manager with real values**
In AWS Console > Secrets Manager > refugee-flow/production, update the secret JSON with real values:
```json
{
  "DATABASE_URL": "postgresql://postgres.xxx:PASSWORD@aws-0-us-east-1.pooler.supabase.com:5432/postgres",
  "ADMIN_SECRET": "your-real-admin-secret",
  "RESEND_API_KEY": "your-real-resend-api-key",
  "ACLED_EMAIL": "",
  "ACLED_PASSWORD": ""
}
```
IMPORTANT: Use Supabase port 5432 (direct/session mode), NOT port 6543 (transaction-only since Feb 2025).

**Step 4: Push initial Docker image to ECR**
```bash
# Get ECR login
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com

# Build and push
docker build -t ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com/refugee-flow:latest .
docker push ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com/refugee-flow:latest
```

**Step 5: Force ECS service to pull the image**
```bash
aws ecs update-service --cluster refugee-flow-cluster --service refugee-flow-service --force-new-deployment --region us-east-1
```

**Step 6: Upload initial frontend to S3**
```bash
npm run build
aws s3 sync dist/ s3://$(aws cloudformation describe-stacks --stack-name refugee-flow --query "Stacks[0].Outputs[?OutputKey=='FrontendBucketName'].OutputValue" --output text) --delete
```

**Step 7: Configure GitHub repository secrets**
In GitHub > Settings > Secrets and variables > Actions, add:
- `AWS_ROLE_ARN`: from CloudFormation output GitHubActionsRoleArn
- `S3_BUCKET`: from CloudFormation output FrontendBucketName
- `CF_DIST_ID`: from CloudFormation output CloudFrontDistributionId

Get all outputs:
```bash
aws cloudformation describe-stacks --stack-name refugee-flow --query "Stacks[0].Outputs" --output table
```
