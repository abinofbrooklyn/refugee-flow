Packages to install (not currently in package.json):
  npm install express-rate-limit cors

express-rate-limit v8 API (use these exact option names — v6 names are deprecated):
  windowMs: 15 * 60 * 1000  // 15 minute window
  max: 200                   // requests per IP per window
  standardHeaders: true      // RateLimit-* headers per RFC
  legacyHeaders: false       // disable X-RateLimit-* (legacy)
  message: { error: 'Rate limit exceeded. Try again in 15 minutes.' }

  DO NOT use: onLimitReached, headers: true, draft_polli_ratelimit_headers (v6 options)

CORS configuration (per locked decision: general internet traffic, not origin-restricted):
  app.use(cors())
  // No origin whitelist. This allows all origins.
  // Optionally restrict to GET only:
  app.use(cors({ methods: ['GET'], allowedHeaders: ['Content-Type'] }))

Middleware order in server.js (CRITICAL — cors and rate limit MUST come before route registration):
  app.use(cors());
  app.use('/data', apiLimiter);
  app.use('/data', dataRoutes);  // existing line — do not move

Integration test pattern for express-rate-limit:
  Use supertest to make N+1 requests in a loop and assert the last one returns 429.
  Adjust the test limiter max to a low number (e.g., 5) for the test so the loop is fast.
  OR: export the limiter factory function and instantiate with a low max in the test.
  Test file location: tests/server/rateLimit.test.js
  Jest is already configured at root — no new test framework setup needed.
