<task type="auto">
  <name>Task 1: Install Vite, create config, move index.html, update package.json scripts</name>
  <files>
    vite.config.js
    index.html
    package.json
    src/components/globe/GlobeVisual.jsx
  </files>
  <action>
    **Step 1 — Install Vite and plugins:**
    ```
    npm install --save-dev vite @vitejs/plugin-react vite-plugin-svgr
    ```

    **Step 2 — Create vite.config.js at project root:**
    ```javascript
    import { defineConfig } from 'vite';
    import react from '@vitejs/plugin-react';
    import svgr from 'vite-plugin-svgr';

    export default defineConfig({
      plugins: [react(), svgr()],
      publicDir: 'src/assets',
      server: {
        port: 8080,
        open: true,
        proxy: {
          '/data': 'http://localhost:2700',
        },
      },
      build: {
        outDir: 'dist',
      },
    });
    ```

    **Step 3 — Create root index.html:**
    Copy `src/index.template.html` to `index.html` at project root. Then:
    - Remove any HtmlWebpackPlugin template syntax (`<%= ... %>` tags) if present
    - Add `<script type="module" src="/src/index.jsx"></script>` before `</body>`
    - Ensure the `<div id="root">` (or whatever the app mount point is) exists

    **Step 4 — Convert remaining require() calls to import:**
    In GlobeVisual.jsx (if not already done in plan 02), convert:
    - `const mousetrap = require('mousetrap')` -> `import mousetrap from 'mousetrap'`
    - `const country_borderLine = require('../../data/countries_states.json')` -> `import country_borderLine from '../../data/countries_states.json'`
    Search for any other `require(` calls in src/ and convert them to ES imports. Vite uses native ESM and `require()` will throw "require is not defined" in the browser.

    **Step 5 — Update package.json scripts:**
    ```json
    "start": "vite",
    "build": "vite build",
    "preview": "vite preview"
    ```
    Keep `nodemon`, `kickoff`, `lint`, and `test` scripts unchanged.
    Remove the `NODE_OPTIONS=--openssl-legacy-provider` prefix — it is no longer needed.

    **Step 6 — Verify Vite build:**
    Run `npx vite build` and confirm it exits 0 with no errors.

    **Step 7 — Verify Vite dev server starts:**
    Run `npx vite` briefly (it should start on port 8080). Kill after confirming startup.

    **IMPORTANT:** Check for any SCSS/CSS imports that might need attention. Vite handles CSS/SCSS natively if `sass` is installed (it is in devDependencies). Verify SCSS imports work by checking the build output.
  </action>
  <verify>
    <automated>npx vite build 2>&1 | tail -10</automated>
  </verify>
  <done>Vite build completes with exit 0. vite.config.js exists. Root index.html exists with module script tag. Package.json scripts updated to use vite. No NODE_OPTIONS hack in any script.</done>
</task>
