<task type="auto">
  <name>Task 2: Migrate GlobeVisual.jsx to BufferGeometry API and replace vendored Octree</name>
  <files>
    src/components/globe/GlobeVisual.jsx

  </files>
  <action>
    This is the core geometry migration. Read GlobeVisual.jsx fully before making changes.

    **Step 1 -- Migrate drawThreeGeo (border lines):**
    Replace all `new THREE.Geometry()` + `vertices.push` with `Float32Array` + `BufferAttribute('position', 3)`. After creating each `THREE.Line` with `LineDashedMaterial`, call `line.computeLineDistances()`.

    **Step 2 -- Migrate addPoint (data point coloring):**
    Replace `geometry.faces[i].color = color` with a vertex color approach. Create a `Float32Array(24 * 3)` for BoxGeometry's 24 vertices. Fill all with the data color. For the top face (vertices 20-23, which were old faces 10-11), set gray (90/255). Use `geometry.setAttribute('color', new THREE.BufferAttribute(colors, 3))`. Material must use `vertexColors: true` instead of `THREE.FaceColors`.

    **Step 3 -- Migrate addPoint (geometry merge):**
    Replace `subgeo.merge(this.point.geometry, this.point.matrix)` with an approach that accumulates positions and colors into growing arrays. Use `this.point.geometry.applyMatrix4(this.point.matrix)` to bake the transform, then read the position/color attributes and append to accumulator arrays. At the end of addData, create a single BufferGeometry from the accumulated arrays. Alternatively, collect individual BoxGeometry instances and use `mergeGeometries` from `three/addons/utils/BufferGeometryUtils.js` -- but only if all geometries have identical attributes (position + color).

    **Step 4 -- Migrate addData (morph targets):**
    Replace `this._baseGeometry = new THREE.Geometry()` with `this._baseGeometry = new THREE.BufferGeometry()`. Replace `this._baseGeometry.morphTargets.push({ name, vertices })` with:
    ```
    if (!this._baseGeometry.morphAttributes.position) {
      this._baseGeometry.morphAttributes.position = [];
    }
    this._baseGeometry.morphAttributes.position.push(new THREE.BufferAttribute(positionsFloat32, 3));
    ```
    Maintain a parallel `this._morphTargetNames = []` array to track count and names.

    **Step 5 -- Migrate createPoints:**
    Replace `_baseGeometry.morphTargets.length` with `this._morphTargetNames.length` (or `this._baseGeometry.morphAttributes.position?.length || 0`). Replace `vertexColors: THREE.FaceColors` with `vertexColors: true`.

    **Step 6 -- applyMatrix -> applyMatrix4:**
    Find and replace both occurrences of `geometry.applyMatrix(` with `geometry.applyMatrix4(`.

    **Step 7 -- Octree replacement:**
    Change the import from `import THREE from '../../THREEJSScript/Octree'` to:
    ```
    import * as THREE from 'three';
    import { Octree } from '@brakebein/threeoctree';
    ```
    Replace `new THREE.Octree({...})` with `new Octree({...})`. The `octree.search()` and `raycaster.intersectOctreeObjects()` APIs remain the same (@brakebein patches Raycaster).

    **Step 8 -- Delete the vendored Octree:**
    Remove `src/THREEJSScript/Octree.js`. If the THREEJSScript directory is now empty, remove it entirely.

    **Step 9 -- Verify build:**
    Run `NODE_OPTIONS=--openssl-legacy-provider npx webpack --env NODE_ENV=production --config webpack/webpack.prod.js` to confirm the app builds with the upgraded three.js. Fix any import or syntax issues.

    **IMPORTANT:** This is a high-risk migration. Read GlobeVisual.jsx thoroughly before making changes. The research patterns in the interfaces block above are your guide, but the exact code structure may require adaptation. When in doubt about a pattern, prefer the simpler replacement that preserves existing behavior.
  </action>
