    This is a large, concentrated migration. All changes are in GlobeVisual.jsx.

    **Step 1 — Install new dependencies:**
    ```
    npm install three@0.165.0
    npm install @brakebein/threeoctree
    ```

    **Step 2 — Read GlobeVisual.jsx fully** and identify every usage of the old API. Expected locations (from research):
    - `drawThreeGeo` method (~line 667): uses `new THREE.Geometry()`, `vertices.push`, creates `THREE.Line`
    - `addPoint` method: creates `BoxGeometry`, sets `geometry.faces[i].color`, calls `geometry.applyMatrix()`, calls `subgeo.merge(this.point.geometry, this.point.matrix)`
    - `addData` method: creates `this._baseGeometry = new THREE.Geometry()`, pushes to `morphTargets` array
    - `createPoints` method: reads `_baseGeometry.morphTargets.length`, creates Mesh with `vertexColors: THREE.FaceColors`
    - `init` method: creates `new THREE.Octree(...)` via the vendored import
    - `raycast_listener`: uses `this.octree.search(...)` and `raycaster.intersectOctreeObjects(...)`
    - Two `geometry.applyMatrix(...)` calls (BoxGeometry translation)
    - `const mousetrap = require('mousetrap')` — convert to import

    **Step 3 — Migrate each pattern:**

    A. **drawThreeGeo (border lines):** Replace all `new THREE.Geometry()` + `vertices.push` with `Float32Array` + `BufferAttribute('position', 3)`. After creating each `THREE.Line` with `LineDashedMaterial`, call `line.computeLineDistances()`.

    B. **addPoint (data point coloring):** Replace `geometry.faces[i].color = color` with a vertex color approach. Create a `Float32Array(24 * 3)` for the BoxGeometry's 24 vertices. Fill all with the data color. For the top face (vertices 20-23, which were old faces 10-11), set gray (90/255). Use `geometry.setAttribute('color', new THREE.BufferAttribute(colors, 3))`. The material must use `vertexColors: true` instead of `THREE.FaceColors`.

    C. **addPoint (geometry merge):** Replace `subgeo.merge(this.point.geometry, this.point.matrix)` with an approach that accumulates positions and colors into growing arrays. Use `this.point.geometry.applyMatrix4(this.point.matrix)` to bake the transform, then read the position/color attributes and append to accumulator arrays. At the end of addData, create a single BufferGeometry from the accumulated arrays. Alternatively, collect individual BoxGeometry instances and use `mergeGeometries` from `three/addons/utils/BufferGeometryUtils.js` — but only if all geometries have identical attributes (position + color).

    D. **addData (morph targets):** Replace `this._baseGeometry = new THREE.Geometry()` with `this._baseGeometry = new THREE.BufferGeometry()`. Replace `this._baseGeometry.morphTargets.push({ name, vertices })` with:
    ```
    if (!this._baseGeometry.morphAttributes.position) {
      this._baseGeometry.morphAttributes.position = [];
    }
    this._baseGeometry.morphAttributes.position.push(new THREE.BufferAttribute(positionsFloat32, 3));
    ```
    Maintain a parallel `this._morphTargetNames = []` array to track count and names.

    E. **createPoints:** Replace `_baseGeometry.morphTargets.length` with `this._morphTargetNames.length` (or `this._baseGeometry.morphAttributes.position?.length || 0`). Replace `vertexColors: THREE.FaceColors` with `vertexColors: true`.

    F. **applyMatrix -> applyMatrix4:** Find and replace both occurrences of `geometry.applyMatrix(` with `geometry.applyMatrix4(`.

    G. **Octree replacement:** Change the import from `import THREE from '../../THREEJSScript/Octree'` to:
    ```
    import * as THREE from 'three';
    import { Octree } from '@brakebein/threeoctree';
    ```
    Replace `new THREE.Octree({...})` with `new Octree({...})`. The `octree.search()` and `raycaster.intersectOctreeObjects()` APIs remain the same (@brakebein patches Raycaster).

    H. **require -> import:** Convert `const mousetrap = require('mousetrap')` to `import mousetrap from 'mousetrap'`. Convert `const country_borderLine = require('../../data/countries_states.json')` to `import country_borderLine from '../../data/countries_states.json'`.

    **Step 4 — Delete the vendored Octree:**
    Remove `src/THREEJSScript/Octree.js` (2141 lines of legacy code). If the THREEJSScript directory is now empty, remove it entirely.

    **Step 5 — Verify build:**
    Run `NODE_OPTIONS=--openssl-legacy-provider npx webpack --env NODE_ENV=production --config webpack/webpack.prod.js` to confirm the app still builds with the upgraded three.js. Fix any import or syntax issues.

    **IMPORTANT:** This is a high-risk migration. Read GlobeVisual.jsx thoroughly before making changes. The research patterns in the interfaces block above are your guide, but the exact code structure may require adaptation. When in doubt about a pattern, prefer the simpler replacement that preserves existing behavior.
