    **Rewrite `src/components/router/config/routeRegistry.jsx`:**

    Convert to v6 element-prop format. Remove the Navbar entry (handled by layout route). Remove the render/children callbacks. Remove isExclusive flag. The registry becomes a simple array of `{ path, element }` objects:

    ```javascript
    import React from 'react';
    import LandingResolver from '../LandingResolver';
    import Conflict from '../../Conflict';
    import RefugeeRoute from '../../RefugeeRoute';
    import About from '../../about/About';
    import AdminPage from '../../Admin/AdminPage';

    const routeRegistry = [
      { path: '/landing', element: <LandingResolver /> },
      { path: '/conflict', element: <Conflict /> },
      { path: '/route/:arg', element: <RefugeeRoute /> },
      { path: '/about', element: <About /> },
      { path: '/admin', element: <AdminPage /> },
    ];

    export default routeRegistry;
    ```

    Note: Router.jsx can either import routeRegistry and map over it, OR inline the routes directly. The locked decision says "keep centralized routeRegistry pattern, adapted for v6" — so keep the registry file but Router.jsx should use it. If using the registry, Router.jsx maps: `routeRegistry.map(r => <Route key={r.path} path={r.path} element={r.element} />)`. The Navbar layout route and catch-all Navigate remain in Router.jsx directly since they are structural, not data-driven.

    Preferred approach (keeps registry meaningful):
    ```jsx
    import routeRegistry from './config/routeRegistry';
    // In Router:
    <Route element={<NavbarLayout />}>
      {routeRegistry.filter(r => r.path !== '/landing').map(r => (
        <Route key={r.path} path={r.path} element={r.element} />
      ))}
    </Route>
    <Route path="/landing" element={routeRegistry.find(r => r.path === '/landing').element} />
    <Route path="*" element={<Navigate to="/landing" replace />} />
    ```

    Remove `import { compact } from 'lodash'` — no longer needed.
    Remove `import MobileDetect from 'mobile-detect'` from routeRegistry — MobileDetect logic is now in LandingResolver.
