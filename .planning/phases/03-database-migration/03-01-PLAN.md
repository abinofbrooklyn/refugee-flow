2. Create `docker-compose.yml` at project root with TWO services:

   **db service** (Postgres):
   ```yaml
   version: '3.9'
   services:
     db:
       image: postgres:16-alpine
       environment:
         POSTGRES_DB: refugeeflow
         POSTGRES_USER: refugeeflow
         POSTGRES_PASSWORD: password
       ports:
         - "5432:5432"
       volumes:
         - pgdata:/var/lib/postgresql/data
       healthcheck:
         test: ["CMD-SHELL", "pg_isready -U refugeeflow -d refugeeflow"]
         interval: 3s
         timeout: 3s
         retries: 10

     setup:
       image: node:20-alpine
       depends_on:
         db:
           condition: service_healthy
       working_dir: /app
       volumes:
         - .:/app
       environment:
         DATABASE_URL: postgresql://refugeeflow:password@db:5432/refugeeflow
       command: sh -c "npm run db:migrate && npm run db:seed"
       profiles: []

   volumes:
     pgdata:
   ```

   CRITICAL (per LOCKED user decision — auto-seed on first run via Docker entrypoint):
   The `setup` service is a Node container that waits for Postgres to be healthy, then runs migrations and seeds automatically. It uses `db` (the Docker service name) as the hostname, NOT `localhost`. The `profiles: []` ensures it runs on every `docker-compose up`. On first run it creates tables and seeds data. On subsequent runs, Knex migrations are idempotent (already-run migrations are skipped) and the seed script (created in Plan 02) is idempotent via TRUNCATE+INSERT.

   NOTE: The `setup` service will exit with an error on Plan 01 execution because `scripts/seed.js` does not exist yet. This is expected. After Plan 02 creates the seed script, `docker-compose down -v && docker-compose up` will auto-migrate AND auto-seed. For Plan 01 verification, run `npm run db:migrate` from the host instead.

3. Create `server/database/migrations/20240101_initial_schema.js`:
   ```javascript
   exports.up = function(knex) {
     return knex.schema
       .createTable('war_events', t => {
         t.integer('id').primary();
         t.smallint('year').notNullable();
         t.string('quarter', 2).notNullable();  // 'q1','q2','q3','q4'
         t.integer('fat');
         t.integer('int');
         t.integer('evt');
         t.jsonb('cot');                          // ["Country","Region"] array
         t.decimal('lat', 5, 2).notNullable();
         t.decimal('lng', 6, 2).notNullable();
         t.index(['year', 'quarter']);
       })
       .createTable('asy_applications', t => {
         t.increments('id').primary();
         t.smallint('year').notNullable();
         t.string('quarter', 2).notNullable();
         t.text('origin').notNullable();
         t.text('destination').notNullable();
         t.integer('value');
         t.integer('record_id');                  // original JSON 'id' field
         t.index(['year', 'quarter']);
       })
       .createTable('route_deaths', t => {
         t.integer('id').primary();
         t.text('date');
         t.text('quarter');                       // "3Q2010" format
         t.smallint('year');
         t.text('dead');                          // stored as TEXT to match API string response
         t.text('missing');
         t.text('dead_and_missing');
         t.text('cause_of_death_display');
         t.text('cause_of_death');
         t.text('location');
         t.text('description');
         t.text('source');
         t.text('lat');                           // TEXT — API returns strings
         t.text('lng');                           // TEXT — API returns strings
         t.text('route');
         t.text('route_display');
         t.text('source_url');
       })
       .createTable('ibc_crossings', t => {
         t.increments('id').primary();
         t.text('route').notNullable();
         t.text('nationality').notNullable();
         t.text('border_location');
         t.smallint('year').notNullable();
         t.string('quarter', 2).notNullable();
         t.integer('value');                      // nullable — source has nulls
         t.index(['route']);
       })
       .createTable('country_routes', t => {
         t.increments('id').primary();
         t.text('country').notNullable();
         t.jsonb('routes').notNullable();         // ["Eastern Mediterranean", ...]
       })
       .createTable('war_notes', t => {
         t.integer('id').primary();
         t.text('notes');
         t.text('source');
       });
   };

   exports.down = function(knex) {
     return knex.schema
       .dropTableIfExists('war_notes')
       .dropTableIfExists('country_routes')
       .dropTableIfExists('ibc_crossings')
       .dropTableIfExists('route_deaths')
       .dropTableIfExists('asy_applications')
       .dropTableIfExists('war_events');
   };
   ```

   Column design rationale:
   - war_events.cot is JSONB to preserve ["Country","Region"] array without JSON.parse on read
   - route_deaths.lat/lng are TEXT because API contract returns string values like "41.24"
   - route_deaths fields (dead, missing, dead_and_missing) are TEXT because source JSON has them as strings
   - ibc_crossings normalizes the dynamic year columns into (route, nationality, year, quarter, value) rows
   - war_notes seeded empty — ACLED population deferred to Phase 4
